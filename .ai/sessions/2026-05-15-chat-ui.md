# Session: Chat UI for Research Debate Engine

**Date:** 2026-05-15

## Goal

Add a web chat UI to the existing Research Debate Engine CLI tool. The UI should look like Claude's chat interface and stream pipeline progress live from the backend.

## What changed

- `debate_engine.py` — `research_debate(topic)` now accepts `on_event=None` async callback. Added `_emit()` and `_run_and_emit()` helpers. Emits 7 event types at key pipeline moments. CLI path unchanged.
- `app.py` (new) — FastAPI server. `POST /api/chat` starts the pipeline as a background task and returns a job ID. `GET /api/stream/{job_id}` streams SSE events from an in-memory `asyncio.Queue`. Serves `static/` as the root.
- `static/index.html` (new) — Single-file vanilla JS + CSS chat UI. Claude-like design. SSE client streams all phases progressively. Tabbed model responses for Phase 1 and Phase 2. Synthesis rendered via marked.js. Input disabled while running.
- `CLAUDE.md` — Added Web UI section, updated dependency list.
- `README.md` — Added Web UI section (run command, project structure), updated install deps.

## Decisions

- Used `asyncio.Queue` as the in-process SSE bridge (no Redis, no disk). Acceptable for single-user local use.
- `_run_and_emit()` wraps each model call so Phase 1/2 responses emit individually as each model finishes, not all at once after `asyncio.gather`.
- `debate()` (quick-debate CLI mode) is not wired to the callback — not needed.
- StaticFiles mount comes after API routes in `app.py` to avoid path conflicts.
- Static dir resolved via `Path(__file__).parent / "static"` to be cwd-independent.
